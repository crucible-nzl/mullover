Sealed-record component reference.

Every component in the Counsel.day design system, rendered live, with the copy-pasteable HTML snippet directly below it. Build new pages by copying from here. Run pwsh ./scripts/brand-verify.ps1 -Path your-new-page.html before commit. The canonical reference is docs/BRAND.md.

component

Records-strip

The top horizontal bar that appears on every page. Plex Mono uppercase, paper background, 1px ink rule beneath. Required on every page; the verifier fails any page without it.

Status pill

Plex Mono uppercase pill with four state variants: sealed (seal background, default for closed records), unsealed (paper-line background, for opened records), pending (transparent, for awaiting state), and live (deep-seal accent, for currently-running decisions).

component

Filing table · the FAQ / Help / Compare replacement

Registry-style table built on native <details> + <summary>. Three-column header: file reference (left), question (centre), revision date and chevron (right). Click any row to expand the answer.

№ 04.07 How is the verdict analysis generated? REV 15 MAY 2026

On a paid decision, the Python analysis layer computes trajectory, trend, stability, sentiment, top themes, and decision-readiness on our own server, then submits the structured result to our AI synthesis tool for a 600-to-1200-word synthesis paragraph.

№ 04.08 Are the votes really sealed until the final day? REV 15 MAY 2026

Yes. The database is designed so neither participant's votes or notes are visible to the other in any form until the verdict is generated. There is no settings panel and no support workflow to override this.

component

Date-stamp opener · replaces the drop cap

Replaces the legacy drop-cap pattern on long-form prose openings. Plex Mono date in seal-clay above the first paragraph, separated by a hairline rule. Set the date via the data-stamp attribute.

On big questions you have pro days and con days, and the difference between Tuesday and Thursday is rarely new information. Counsel.day captures the whole stream, not just tonight's mood. 

<p class="opener" data-stamp="14 MAY 2026">
  Body text on a long-form essay; the date stamp positions above
  via a CSS ::before pseudo-element. The opener replaces the
  legacy drop-cap pattern entirely.
</p>

verifier

Pre-commit check

Run the verifier on a single new file before you commit. It returns exit code 0 on success, 1 on any failure. The full ruleset is documented at docs/BRAND.md 12.

# Whole-repo lint (run before push)
pwsh ./scripts/brand-verify.ps1

# Single-file lint (run before commit)
pwsh ./scripts/brand-verify.ps1 -Path ./your-new-page.html

# Verbose mode (show every passed check)
pwsh ./scripts/brand-verify.ps1 -Verbose
Counsel.day

A decision tool for couples and families, for the questions a single conversation cannot resolve. Each evening, you and your partner cast one private vote on the same question, for a duration you both set, with every vote sealed in the database until the final day. When the verdict opens, you see where you converged, where you diverged, the themes underneath your own words, and one specific next conversation. The deciding stays with you; the structure is ours. Worldwide, priced in USD.
